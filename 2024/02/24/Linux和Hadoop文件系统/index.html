# Linux 文件系统与 Hadoop 文件系统

Hadoop 需要部署在 Linux 系统上，那这两者的文件系统有什么区别呢？

# 文件目录

以下是 LInux 文件系统的文件目录：

Linux 顶级目录为根目录 /。其它文件的含义见下：

  • bin: 二进制文件，用于存储可执行文件，包含 linux 系统指令，如 cd,mkdir 等。

  • lib: 存放库文件，Linux 库文件，如 python。

  • etc: 配置文件，如环境变量配置文件 profile。

  • home：用户空间，如 user1,user2。

  • root：根用户目录，只有管理员才有权限进入。

  • usr: 存放程序和文件。

  • dev: 设备文件目录，如网卡声卡。

  • var: 存放系统经常变化的文件。

  • temp：存放临时文件

  • proc: 虚拟目录

其中常用的文件目录为前 6 个。

接下来看 Hadoop 目录：

  • bin：可执行文件，存放 hadoop 管理脚本指令，如 hadoop,hdfs。

  • sbin：提供用户管理指令，如 start-yarn.sh，stop-yarn.sh。

  • etc：配置文件，如 hadoop-env.sh，core-site.xml。

  • share：各模块编译产生的 jar 包，如 hadoop-hdfs-nfs.jar。

  • lib：运行可能需要的库。

  • linexec：各个服务的 shell 配置文件。

  • logs：日志文件。

  • include：一些可能需要的头文件。

  • temp：临时文件。

我们再了解下 JDK 的文件目录：

  • bin: 可执行文件，如 javac,java
  • lib：库文件
  • jre：运行环境
  • db：关系数据库
  • include：头文件

# 关于环境变量

环境变量的设置是为了方便系统再执行指令时能够找到指令（寻址）和找到依赖的库（支撑）。

所以设置环境变量一般要设置以下三个变量：

  • HOME 变量：即框架或服务安装的目录，如 JAVA_HOME 和 HADOOP_HOME。
  • PATH 变量：告诉系统执行指令时寻找的路径，一般在 bin 目录，如JAVAHOME/bin,JAVA_HOME/bin,JAVAH​OME/bin,HADOOP_HOME/bin。
  • CLASSPTHN 变量：类路径，系统程序支撑的库或依赖，一般在 lib 目录。

# 常用指令

Linux 和 Hadoop 指令都有很多，但常用到的就那几种。就像几万汉字里日常也就用到几千。因此我们总结一些常用指令：

# Linux

  • 查看路径：pwd
  • 切换路径：cd
  • 创建文件：touch/vim/cat
  • 删除文件：rm
  • 创建文件夹：mkdir
  • 删除文件夹：rm -r
  • 编辑文件：vi/vim/nano/gredit
  • 解压和压缩：tar/bzip2/gzip
  • 移动文件：mv
  • 复制文件：cp
  • 删除文件：rm
  • 下载文件：apt-install
  • 切换权限：sudo
  • 切换用户：su

Hadoop

  • 创建文件夹：hadoop fs -mkdir
  • 查看目录：hadoop fs -ls
  • 移动 / 复制 / 删除：hadoop fs -mv/cp/rm
  • 从本地上传文件：hadoop fs -put
  • 下载文件到本地：hadoop fs -get
  • 运行程序：hadoop jar

由于 hadoop 自身分布式特性，因此不擅长文件创建与编辑。

实际需要 Linux 本地文件系统与 Hadoop 文件系统相互配合使用。
