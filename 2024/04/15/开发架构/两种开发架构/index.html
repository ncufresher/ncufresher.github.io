# 开发架构

在计算中所有的应用程序常常基于以下两种开发架构：

  • C/S 架构（Client-Server）：客户端 - 服务器架构，这里的客户端显著的特点是需要下载，即安装在本地的应用，当使用这些软件时，常常需要与后端服务器进行数据连接。

  • B/S 架构（Brower-Server）：浏览器 - 服务器架构，像日常常见的 WEB 程序，网站等，都是通过浏览器与后端服务器连接。

C/S 架构由于将软件安装在本地，因此在访问和运行过程中会更加稳定，但如果软件太大是便会占用较多的本地空间，而 B/S 架构于是显得轻量，当然也可以通过云计算模式将将服务在云端进行。

当从这两个基本开发框架出发去看其他应用开发的过程时，便就更加清楚每一步的作用。

# 举例

在使用 Hexo+Github 搭建个人博客时，搜索到的大部分是教你每一步怎么做怎么做，只要无脑按部就班便就能完成了，但是为什么是这样？

而到搜索具体的搭建的原理时的资料却是少之又少，从 B/S 架构来理解便就清楚多了。

Hexo 是一个静态网页框架，而 Github 提供了一个文件托管平台的作用，git 工具可以帮助我们从本地管理 Github 仓库，于是我们在使用浏览器时访问的其实是 github 的文件仓库，然后在本地使用 hexo 工具进行网页的管理与博客内容的上传，最终在 github 得到内容的更新，于是不同的内容便展现在网页端。
